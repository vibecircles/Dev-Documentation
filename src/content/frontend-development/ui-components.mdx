# UI Components

UI components provide the visual interface and user interactions throughout the application.

## 1. LogoSearch

**Location:** `src/Components/LogoSearch/LogoSearch.js`

### Purpose
Displays the application logo and provides search functionality.

### Features
- Application logo display
- Search input field
- Navigation element

### Usage
Typically used in headers or navigation bars.

## UI Component Patterns

### Modal Components
Both `ShareModal` and `ProfileModal` follow modal patterns:
- Overlay background
- Centered content
- Close functionality
- Form handling (for ProfileModal)

### Container Components
Components like `RightSide` act as containers:
- Wrap multiple child components
- Provide consistent layout
- Manage spacing and styling

### Card Components
Components like `InfoCard` use card design:
- Elevated appearance
- Rounded corners
- Padding and spacing
- Visual hierarchy

## Styling Approach

UI components use:
- **CSS Modules:** Component-specific styles
- **CSS Variables:** For theming (colors, spacing)
- **Material-UI Icons:** For consistent iconography
- **Flexbox/Grid:** For layouts

## Component Props

### Common Props Pattern
```javascript
// Container component
<RightSide />

// Card component with data
<InfoCard data={infoData} />

// Modal component with handlers
<ProfileModal 
  isOpen={isOpen}
  onClose={handleClose}
  onSubmit={handleSubmit}
/>
```

## Accessibility Considerations

UI components should:
- Use semantic HTML
- Include proper ARIA labels
- Support keyboard navigation
- Maintain focus management (especially modals)

## Responsive Design

UI components are designed to be responsive:
- Adapt to different screen sizes
- Use flexible layouts (Flexbox/Grid)
- Hide/show elements based on viewport
- Maintain usability on mobile devices
