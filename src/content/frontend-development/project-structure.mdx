## Directory Descriptions

### `/src/actions`
Contains Redux action creators that dispatch actions to update the application state. Each file handles actions for a specific domain (Auth, Posts, Uploads, Users).

### `/src/api`
Centralized API request handlers using Axios. These functions make HTTP requests to the backend API and return promises.

### `/src/Components`
Reusable UI components organized by feature:
- **Post Components:** Post, Posts, PostShare, PostSide
- **Profile Components:** ProfileCard, ProfileModal, ProfilePageLeft, profileSide
- **Social Components:** FollowersCard, TrendCard, UserFollow
- **UI Components:** InfoCard, LogoSearch, RightSide

### `/src/Pages`
Top-level page components that represent different routes:
- **Auth:** Login and Signup page
- **Home:** Main feed page
- **Profile:** User profile page

### `/src/reducers`
Redux reducers that specify how the application state changes in response to actions.

### `/src/store`
Redux store configuration with middleware (Redux Thunk) and persistence to localStorage.

## Component Organization

Components are organized by feature/domain:
- Each component has its own folder
- Components include both `.js` and `.css` files
- Related components are grouped together (e.g., Post-related components)
